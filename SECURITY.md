# Security

## Current Status

The current version of this software is designed to be run on local, trusted networks. It has not yet been hardened for public internet deployment.

When future versions are developed for multi-tenant and internet-accessible use, this document will be updated with more comprehensive security guidance.

## Reporting Vulnerabilities

If you discover a security vulnerability in Trip Tools, please report it responsibly:

**Email:** [info@pomdp.com](mailto:info@pomdp.com)

**What to include:**
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

**What to expect:**
- We will acknowledge receipt within 48 hours
- We will investigate and provide an initial assessment within 7 days
- We will work with you to understand and resolve the issue
- We will credit reporters in release notes (unless you prefer to remain anonymous)

**Please do not:**
- Publicly disclose the vulnerability before we've had a chance to address it
- Access or modify other users' data
- Perform actions that could harm the service or its users
